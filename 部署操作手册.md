# 茶颜同比店杯数达成看板 — 部署操作手册

> 适用场景：将单文件 HTML 看板部署为永久链接，支持自动数据刷新。

---

## 一、整体架构

```
┌─────────────────────────────────────────────┐
│  本地电脑                                    │
│  ├─ 下载 xlsx 数据源 → Download 文件夹       │
│  ├─ watch_xlsx.py 监控新文件 → 自动转换       │
│  ├─ auto_refresh.py 转 CSV → 更新 JSON → 推送 │
│  └─ 定时任务 每天 9:20 兜底执行              │
└────────────────┬────────────────────────────┘
                 │ git push
                 ▼
┌─────────────────────────────────────────────┐
│  GitHub (summerrain620/waterbar)             │
│  ├─ index.html       看板网页                │
│  ├─ 看板数据.json    数据源                  │
│  └─ auto_refresh.py  刷新脚本               │
└────────────────┬────────────────────────────┘
                 │ GitHub Pages
                 ▼
         https://你的用户名.github.io/仓库名/
```

## 三、部署到 GitHub Pages

### 步骤 1：注册 GitHub 账号

访问 https://github.com/signup，用邮箱注册，设置用户名。

> ⚠️ 用户名决定永久链接格式：`https://你的用户名.github.io/`

### 步骤 2：创建仓库

1. 访问 https://github.com/new
2. 填写 Repository name（如 `waterbar`）
3. 选择 **Public**（免费版 Pages 要求公开仓库）
4. **不要**勾选 Add a README / .gitignore / license
5. 点击 Create repository

### 步骤 3：创建 Personal Access Token

1. 访问 https://github.com/settings/tokens
2. 点击 Generate new token (classic)
3. Note 填写 `waterbar`，Expiration 选 7 days
4. 勾选 `repo` 权限
5. 点击 Generate token
6. **复制 token**（只显示一次，妥善保存）

### 步骤 4：推送代码

在项目目录下打开命令提示符，依次执行：

```cmd
cd C:\Users\你的用户名\waterbar

rem 初始化 Git
git init -b main
git add -A
git commit -m "初始化看板"

rem 推送（把下面 YOUR_TOKEN 换成你的 token）
git push https://你的用户名:YOUR_TOKEN@github.com/你的用户名/waterbar.git main
```

### 步骤 5：启用 GitHub Pages

推送成功后，用 API 启用 Pages（把 YOUR_TOKEN 和 YOUR_USERNAME 换成你的）：

```cmd
curl -X POST ^
  -H "Authorization: token YOUR_TOKEN" ^
  -H "Accept: application/vnd.github+json" ^
  https://api.github.com/repos/YOUR_USERNAME/waterbar/pages ^
  -d "{\"source\":{\"branch\":\"main\",\"path\":\"/\"}}"
```

如果系统没有 curl，可以直接在浏览器里操作：
1. 打开仓库页面 → Settings → Pages
2. Source 选 Deploy from a branch
3. Branch 选 main，文件夹选 / (root)
4. 点击 Save

**部署生效需 1-3 分钟**，访问 `https://你的用户名.github.io/仓库名/` 查看。

## 四、配置自动刷新

### 方案 A：文件实时监控（推荐）

只要把 xlsx 下载到 Downloads 文件夹，系统自动识别并推送。

#### 1. 创建开机自启动任务

以管理员身份打开 PowerShell（Win → 搜索 PowerShell → 右键 → 以管理员身份运行），执行：

```powershell
schtasks /create /tn "DashboardWatch" /tr "'C:\Users\你的用户名\.workbuddy\binaries\python\envs\default\Scripts\pythonw.exe' 'C:\Users\你的用户名\waterbar\watch_xlsx.py'" /sc onlogon /delay 0000:30
```

作用：每次开机后 30 秒，自动启动后台监控进程。

#### 2. 启动监控

```powershell
Start-Process -WindowStyle Hidden -FilePath "C:\Users\你的用户名\.workbuddy\binaries\python\envs\default\Scripts\pythonw.exe" -ArgumentList "C:\Users\你的用户名\waterbar\watch_xlsx.py"
```

没有窗口弹出是正常的，监控已在后台运行。

#### 3. 验证监控是否运行

```powershell
Get-Process pythonw
```

看到 `pythonw` 进程即表示成功。

### 方案 B：每日定时刷新（兜底）

担心监控进程意外退出？再加一道定时任务作为兜底：

```powershell
schtasks /create /tn "DashboardDailyRefresh" /tr "'C:\Users\你的用户名\.workbuddy\binaries\python\envs\default\Scripts\python.exe' 'C:\Users\你的用户名\waterbar\auto_refresh.py'" /sc daily /st 09:20
```

作用：每天 9:20 自动执行一次刷新（无论监控是否在运行）。

| 时间 | 谁做 | 做什么 |

## 六、修改看板功能后如何更新

当你需要改看板代码（调整布局、新增图表等），修改完 `index.html` 后，通过 GitHub API 推上去：

```python
import base64, json, urllib.request

token = "YOUR_TOKEN"
username = "YOUR_USERNAME"
repo = "waterbar"

# 读取当前文件 SHA
url = f"https://api.github.com/repos/{username}/{repo}/contents/index.html"
req = urllib.request.Request(url, headers={
    "Authorization": f"token {token}",
    "Accept": "application/vnd.github+json"
})
with urllib.request.urlopen(req) as resp:
    sha = json.loads(resp.read())["sha"]

# 上传修改后的文件
with open("C:/Users/你的用户名/waterbar/index.html", "r", encoding="utf-8") as f:
    content = f.read()
encoded = base64.b64encode(content.encode("utf-8")).decode("ascii")
payload = json.dumps({
    "message": "更新看板功能",
    "content": encoded,
    "sha": sha,
    "branch": "main"
}).encode("utf-8")

req2 = urllib.request.Request(url, data=payload, headers={
    "Authorization": f"token {token}",
    "Accept": "application/vnd.github+json",
    "Content-Type": "application/json"
}, method="PUT")
with urllib.request.urlopen(req2) as resp:
    print("推送成功")
```

## 七、常见问题

### Q：国内打不开 GitHub Pages？

更换 DNS 为 `114.114.114.114` 或使用 VPN/代理。

### Q：监控没反应？

查看日志文件 `C:\Users\你的用户名\waterbar\auto_refresh.log`，里面记录了每次处理的详细情况。

### Q：电脑关机了还会刷新吗？

不会。监控和定时任务都依赖电脑开机。关机期间的数据会在下次开机后由定时任务补上。

### Q：Token 过期了怎么办？

1. 重新创建 token（步骤 3）
2. 更新 `auto_refresh.py` 和 `watch_xlsx.py` 中的 `GITHUB_TOKEN`
3. 更新 `refresh.bat` 中的 token

### Q：换电脑了怎么办？

1. 在新电脑上安装 Git + Python + 依赖包
2. 从 GitHub 克隆仓库：`git clone https://github.com/你的用户名/waterbar.git`
3. 重新执行「第四章」配置自动刷新
